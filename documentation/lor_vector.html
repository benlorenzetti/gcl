lor_vector.h

Named like C++, a lor_vector container is a dynamic array that grows and shrinks automatically as elements are added and removed. Elements are stored sequentially in a contiguous block of memory, basically an array that is managed on the heap. Lor vector is a part of a larger generic container library allowing "objecty" programming in C with functions analogous to those in the C++ STL.

Accessing an element is O(1) fast because its position is the beginning of the array plus an offset based on the type size. Adding or removing elements at the back is usually O(1) fast unless an interal call to realloc() must be made. However, insertions are O(n) costly because all elements after the insertion point have to be copy shifted rightwards.

Any struct can be stored after its size is registered during initialization. For structures with pointers to other memory, deep copying is supported after registering copy constructor and destructor functions during initialazation.

For large n, memory is allocated exponetially in powers of 1.3 (this can be adjusted by redefining LOR_VECTOR_A in lor_vector.h. For small n, each reallocation adds space for at least LOR_VECTOR_B new elements.

Library namespace

All exterally visible types, functions, and macros are prefixed with "lor_vector_" or "LOR_VECTOR_" to avoid namespace collisions. If this make your code too verbose, the optional LOR_VECTOR_NAMESPACE can be defined.

#include <stdio.h>
#include "lor_vector.h"

int main() {
  lor_vector_t prime_numbers = LOR_VECTOR(int, NULL, NULL);
  lor_vector_push_back(&prime_numbers, (void*)2);
  lor_vector_push_back(&prime_numbers, (void*)3);
  printf("vector size = %d\n", lor_vector_size(&prime_numbers));
} #include <stdio.h>
#define LOR_VECTOR_NAMESPACE vec
#include "lor_vector.h"

int main() {
  vec_t prime_numbers = LOR_VECTOR_INIT(int, NULL, NULL);
  vec.push_back(&prime_numbers, (void*)2);
  vec.push_back(&prime_numbers, (void*)3);
  printf("vector size = %d\n", vec.size(&prime_numbers));
}
